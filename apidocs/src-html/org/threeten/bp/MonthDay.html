060/**
061 * A month-day in the ISO-8601 calendar system, such as {@code --12-03}.
062 * <p>
063 * {@code MonthDay} is an immutable date-time object that represents the combination
064 * of a year and month. Any field that can be derived from a month and day, such as
065 * quarter-of-year, can be obtained.
066 * <p>
067 * This class does not store or represent a year, time or time-zone.
068 * For example, the value "December 3rd" can be stored in a {@code MonthDay}.
069 * <p>
070 * Since a {@code MonthDay} does not possess a year, the leap day of
071 * February 29th is considered valid.
072 * <p>
073 * This class implements {@link TemporalAccessor} rather than {@link Temporal}.
074 * This is because it is not possible to define whether February 29th is valid or not
075 * without external information, preventing the implementation of plus/minus.
076 * Related to this, {@code MonthDay} only provides access to query and set the fields
077 * {@code MONTH_OF_YEAR} and {@code DAY_OF_MONTH}.
078 * <p>
079 * The ISO-8601 calendar system is the modern civil calendar system used today
080 * in most of the world. It is equivalent to the proleptic Gregorian calendar
081 * system, in which today's rules for leap years are applied for all time.
082 * For most applications written today, the ISO-8601 rules are entirely suitable.
083 * However, any application that makes use of historical dates, and requires them
084 * to be accurate will find the ISO-8601 approach unsuitable.
085 *
086 * <h3>Specification for implementors</h3>
087 * This class is immutable and thread-safe.
088 */
089public final class MonthDay
090        extends DefaultInterfaceTemporalAccessor
091        implements TemporalAccessor, TemporalAdjuster, Comparable<MonthDay>, Serializable {

164    /**
165     * Obtains an instance of {@code MonthDay}.
166     * <p>
167     * The day-of-month must be valid for the month within a leap year.
168     * Hence, for February, day 29 is valid.
169     * <p>
170     * For example, passing in April and day 31 will throw an exception, as
171     * there can never be April 31st in any year. By contrast, passing in
172     * February 29th is permitted, as that month-day can sometimes be valid.
173     *
174     * @param month  the month-of-year to represent, not null
175     * @param dayOfMonth  the day-of-month to represent, from 1 to 31
176     * @return the month-day, not null
177     * @throws DateTimeException if the value of any field is out of range
178     * @throws DateTimeException if the day-of-month is invalid for the month
179     */
180    public static MonthDay of(Month month, int dayOfMonth) {
181        Objects.requireNonNull(month, "month");
182        DAY_OF_MONTH.checkValidValue(dayOfMonth);
183        if (dayOfMonth > month.maxLength()) {
184            throw new DateTimeException("Illegal value for DayOfMonth field, value " + dayOfMonth +
185                    " is not valid for month " + month.name());
186        }
187        return new MonthDay(month.getValue(), dayOfMonth);
188    }

191     * Obtains an instance of {@code MonthDay}.
192     * <p>
193     * The day-of-month must be valid for the month within a leap year.
194     * Hence, for month 2 (February), day 29 is valid.
195     * <p>
196     * For example, passing in month 4 (April) and day 31 will throw an exception, as
197     * there can never be April 31st in any year. By contrast, passing in
198     * February 29th is permitted, as that month-day can sometimes be valid.
199     *
200     * @param month  the month-of-year to represent, from 1 (January) to 12 (December)
201     * @param dayOfMonth  the day-of-month to represent, from 1 to 31
202     * @return the month-day, not null
203     * @throws DateTimeException if the value of any field is out of range
204     * @throws DateTimeException if the day-of-month is invalid for the month
205     */
206    public static MonthDay of(int month, int dayOfMonth) {
207        return of(Month.of(month), dayOfMonth);
208    }

211    /**
212     * Obtains an instance of {@code MonthDay} from a temporal object.
213     * <p>
214     * A {@code TemporalAccessor} represents some form of date and time information.
215     * This factory converts the arbitrary temporal object to an instance of {@code MonthDay}.
216     * <p>
217     * The conversion extracts the {@link ChronoField#MONTH_OF_YEAR MONTH_OF_YEAR} and
218     * {@link ChronoField#DAY_OF_MONTH DAY_OF_MONTH} fields.
219     * The extraction is only permitted if the date-time has an ISO chronology.
220     * <p>
221     * This method matches the signature of the functional interface {@link TemporalQuery}
222     * allowing it to be used in queries via method reference, {@code MonthDay::from}.
223     *
224     * @param temporal  the temporal object to convert, not null
225     * @return the month-day, not null
226     * @throws DateTimeException if unable to convert to a {@code MonthDay}
227     */
228    public static MonthDay from(TemporalAccessor temporal) {
229        if (temporal instanceof MonthDay) {
230            return (MonthDay) temporal;
231        }
232        try {
233            if (IsoChronology.INSTANCE.equals(Chronology.from(temporal)) == false) {
234                temporal = LocalDate.from(temporal);
235            }
236            return of(temporal.get(MONTH_OF_YEAR), temporal.get(DAY_OF_MONTH));
237        } catch (DateTimeException ex) {
238            throw new DateTimeException("Unable to obtain MonthDay from TemporalAccessor: " + temporal.getClass(), ex);
239        }
240    }

285    /**
286     * Checks if the specified field is supported.
287     * <p>
288     * This checks if this month-day can be queried for the specified field.
289     * If false, then calling the {@link #range(TemporalField) range} and
290     * {@link #get(TemporalField) get} methods will throw an exception.
291     * <p>
292     * If the field is a {@link ChronoField} then the query is implemented here.
293     * The {@link #isSupported(TemporalField) supported fields} will return valid
294     * values based on this date-time.
295     * The supported fields are:
296     * <ul>
297     * <li>{@code MONTH_OF_YEAR}
298     * <li>{@code YEAR}
299     * </ul>
300     * All other {@code ChronoField} instances will return false.
301     * <p>
302     * If the field is not a {@code ChronoField}, then the result of this method
303     * is obtained by invoking {@code TemporalField.isSupportedBy(TemporalAccessor)}
304     * passing {@code this} as the argument.
305     * Whether the field is supported is determined by the field.
306     *
307     * @param field  the field to check, null returns false
308     * @return true if the field is supported on this month-day, false if not
309     */
310    @Override
311    public boolean isSupported(TemporalField field) {
312        if (field instanceof ChronoField) {
313            return field == MONTH_OF_YEAR || field == DAY_OF_MONTH;
314        }
315        return field != null && field.isSupportedBy(this);
316    }

318    /**
319     * Gets the range of valid values for the specified field.
320     * <p>
321     * The range object expresses the minimum and maximum valid values for a field.
322     * This month-day is used to enhance the accuracy of the returned range.
323     * If it is not possible to return the range, because the field is not supported
324     * or for some other reason, an exception is thrown.
325     * <p>
326     * If the field is a {@link ChronoField} then the query is implemented here.
327     * The {@link #isSupported(TemporalField) supported fields} will return
328     * appropriate range instances.
329     * All other {@code ChronoField} instances will throw a {@code DateTimeException}.
330     * <p>
331     * If the field is not a {@code ChronoField}, then the result of this method
332     * is obtained by invoking {@code TemporalField.rangeRefinedBy(TemporalAccessor)}
333     * passing {@code this} as the argument.
334     * Whether the range can be obtained is determined by the field.
335     *
336     * @param field  the field to query the range for, not null
337     * @return the range of valid values for the field, not null
338     * @throws DateTimeException if the range for the field cannot be obtained
339     */
340    @Override
341    public ValueRange range(TemporalField field) {
342        if (field == MONTH_OF_YEAR) {
343            return field.range();
344        } else if (field == DAY_OF_MONTH) {
345            return ValueRange.of(1, getMonth().minLength(), getMonth().maxLength());
346        }
347        return super.range(field);
348    }

400    @Override
401    public long getLong(TemporalField field) {
402        if (field instanceof ChronoField) {
403            switch ((ChronoField) field) {
404                // alignedDOW and alignedWOM not supported because they cannot be set in with()
405                case DAY_OF_MONTH: return day;
406                case MONTH_OF_YEAR: return month;
407            }
408            throw new DateTimeException("Unsupported field: " + field.getName());
409        }
410        return field.getFrom(this);
411    }

488    * Returns a copy of this {@code MonthDay} with the month-of-year altered.
489    * <p>
490    * This returns a month-day with the specified month.
491    * If the day-of-month is invalid for the specified month, the day will
492    * be adjusted to the last valid day-of-month.
493    * <p>
494    * This instance is immutable and unaffected by this method call.

499    public MonthDay with(Month month) {
500        Objects.requireNonNull(month, "month");
501        if (month.getValue() == this.month) {
502            return this;
503        }
504        int day = Math.min(this.day, month.maxLength());
505        return new MonthDay(month.getValue(), day);
506    }

556    /**
557     * Adjusts the specified temporal object to have this month-day.
558     * <p>
559     * This returns a temporal object of the same observable type as the input
560     * with the month and day-of-month changed to be the same as this.
561     * <p>
562     * The adjustment is equivalent to using {@link Temporal#with(TemporalField, long)}
563     * twice, passing {@link ChronoField#MONTH_OF_YEAR} and
564     * {@link ChronoField#DAY_OF_MONTH} as the fields.
565     * If the specified temporal object does not use the ISO calendar system then
566     * a {@code DateTimeException} is thrown.
567     * <p>
568     * In most cases, it is clearer to reverse the calling pattern by using
569     * {@link Temporal#with(TemporalAdjuster)}:
570     * <pre>
571     *   // these two lines are equivalent, but the second approach is recommended
572     *   temporal = thisMonthDay.adjustInto(temporal);
573     *   temporal = temporal.with(thisMonthDay);
574     * </pre>
575     * <p>
576     * This instance is immutable and unaffected by this method call.
577     *
578     * @param temporal  the target object to be adjusted, not null
579     * @return the adjusted object, not null
580     * @throws DateTimeException if unable to make the adjustment
581     * @throws ArithmeticException if numeric overflow occurs
582     */
583    @Override
584    public Temporal adjustInto(Temporal temporal) {
585        if (Chronology.from(temporal).equals(IsoChronology.INSTANCE) == false) {
586            throw new DateTimeException("Adjustment only supported on ISO date-time");
587        }
588        temporal = temporal.with(MONTH_OF_YEAR, month);
589        return temporal.with(DAY_OF_MONTH, Math.min(temporal.range(DAY_OF_MONTH).getMaximum(), day));
590    }

593    /**
594     * Combines this month-day with a year to create a {@code LocalDate}.
595     * <p>
596     * This returns a {@code LocalDate} formed from this month-day and the specified year.
597     * <p>
598     * A month-day of February 29th will be adjusted to February 28th in the resulting
599     * date if the year is not a leap year.
600     * <p>
601     * This instance is immutable and unaffected by this method call.
602     *
603     * @param year  the year to use, from MIN_YEAR to MAX_YEAR
604     * @return the local date formed from this month-day and the specified year, not null
605     * @throws DateTimeException if the year is outside the valid range of years
606     */
607    public LocalDate atYear(int year) {
608        return LocalDate.of(year, month, isValidYear(year) ? day : 28);

614     * <p>
615     * The comparison is based first on value of the month, then on the value of the day.
616     * It is "consistent with equals", as defined by {@link Comparable}.
617     *
618     * @param other  the other month-day to compare to, not null
619     * @return the comparator value, negative if less, positive if greater

621    public int compareTo(MonthDay other) {
622        int cmp = (month - other.month);
623        if (cmp == 0) {
